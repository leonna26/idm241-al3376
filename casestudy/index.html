Search Feature Microinteraction
Case Study - Anna Leong

Overview | Context and Challenges | Process and Insights | The Solution | The Results

Overview

This case study outlines a project completed for IDM 241: Microinteractions. The project was broken into three parts - a simple, medium, and complex microinteraction. I chose to develop a microinteraction for a search feature in order to create a more engaging, interactive experience. HTML, CSS, and JavaScript were used in the making of this project. Through personal research looking into resources, guidance from professor, classmates, and friends, I was able to strengthen my knowledge and proficiency of these languages in order to achieve the goals and plans I had intended.

Context and Challenge

Background:
This project was produced as an individual project for a class - specifically IDM 241; Microinteractions. The class teaches students to build upon HTML, CSS, and JavaScript skills to create a microinteraction(s).

Description:
What classifies as a microinteraction? A microinteraction includes four-five elements: a trigger(s), rule(s), feedback, and loops/modes. For my project, I decided to create a microinteraction that would animate a search feature based on inspiration I found from Dribble.com. Upon clicking the search icon, the icon will lower itself and then open horizontally into a search bar with a text input field. After that, the user will be able to click the icon again to return the icon to its original position. This is a microinteraction that I hope to sometime implement and put into a website of my own. The project as a whole was split into three parts: simple, medium, and complex microinteraction.

Timeline:
The timeline for the whole duration of this project was the course of ten weeks. There were various stages of the process - including an alpha, beta, and final. These all culminated into the whole project.

Purpose:
The purpose of this project was to enhance an element of an interaction, specifically the search function, of a website. This project also hoped to build upon and expand pre-existing knowledge of HTML, CSS, and JavaScript.

The Problem:
So far, my personal website does not have a search function. My project for IDM232 uses a search function, but it is currently not interactive. Creating a microinteraction like this would give me something to implement in my websites that is engaging and interactive to my users instead of just remaining static.

Goals & Objectives:
The goals of this project were to create a fully functioning interactive search feature. The objectives include: users being able to click the search icon, the search text field opening horizontally, and the icon being able to close and return to its original position.

Process and Insights

For the design process, I drew inspiration from Dribble. Here is the source to the reference that I wanted to build: Source example
The process of this project was divided up into three parts: simple, medium, and complex microinteraction to create the final result.


1. Simple Microinteraction: for this microinteraction, the goal was for the user to be able to click the search icon and have the icon bounce down once clicked. This project forced me to have a refresher on JavaScript. I tried to research and look up alternative ways to implement an onclick function without having to use JavaScript, but finally gave in. It was a good review to need to use it. One of the biggest problems that I encountered when making this simple microinteraction was how to get the animation to bounce down properly. There was always something wrong with the trigger where the trigger click would only work once the page is refreshed, and it would not have the bouncy animation effect I desired. fiAfter researching and trying to find ways to troubleshoot, I tried switching the animation name from "bounceInDown" to "bounceOutDown" and this helped the animation out a lot. At first, the animation swooped down but in doing so it was too fast and sort of disappeared/faed in the process. In the end, the bounceOutDown function somehow worked and produced a smooth drop to the icon animation. Here is the GIF for the Simple Microinteraction Build reference.


2. Medium Microinteraction: for this microinteraction, users can hover over the search icon and a search bar with a text input field opens horizontally. For this project, I looked for reference code in CodePen.io to find similar examples of a search bar expanding. I added hover states to classes so that it triggered what I wanted the element to do when a user hovered over it. The two frustrations I had for this part of the project was trying to figure out to get the "Search" text to display over the search bar instead of below it. I could not figure it out for the longest time! Finally, I tried margin-top with a negative number instead of positive. And it finally worked! Another huge frustration was trying to get the search icon to stay within the search bar once it was expanded. When the screen or browser width was resized to be smaller, the search icon would clip down off of the search bar content. With the help of several media queries for the search bar, I was able to adjust the width of the search bar. This solved the problem of having content be too wide and therefore not being contained in the search bar. Looking back, I feel like this microinteraction in itself was not hard to put together. Instead, all the power held within the mind - of trying to troubleshoot, knowing what to search for to utilize sample code, and executing/manipulating the reference code into something that is what I desire. Here is the GIF that visualizes this concept for the Medium Microinteraction Build.


3. Complex Microinteraction: for this microinteraction, the goal is for users to be able to click the search icon to drop down, then open horizontally to enter text into a text field. The Results

Overall, this project came out to be a success. I was able to implement the elements I had planned prior to the build process for each part of the project. I was able to cement my knowledge of JavaScript, as well as explore different CSS animation functions. I was also able to grow better at using media queries, since I had several problems getting my site responsive enough or format correctly at all times. Comparing my alpha and beta builds, the search icon is differnt between the two. In the future, I would have stuck with the same one or adjusted the other accordingly. I would also adjust the opacity or "drop" of the animation in the alpha build. Although it is fully functioning, if I were to refine it more, I would not have the slight hiccip of the search icon bouncing up for a fraction of a second before dropping down. I also notice that in the reference GIF there is not so much variation to the search icon itself when it is bouncing down. It appears that there is some opacity difference as the search icon comes down, however that is not the case in the beta build. Therefore with those things said, I would try to get the icon as smooth and seemless as possible, refining the animation and the opacity of the element.
